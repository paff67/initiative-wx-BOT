# Product

## Register

product

## Users

Observer is used by the operator and developer of the Presence Kernel. They are usually checking a live Weixin proactive system from a desktop browser, often while debugging prompts, world collectors, delivery results, proxy failures, and profile configuration.

## Product Purpose

The console exists to make a complex proactive agent observable without forcing the operator to read raw event files. Success means the operator can quickly answer: what state did the kernel infer, why did it decide to send or stay silent, which world signals were used, what would be delivered, and whether proxy or delivery failures need attention.

## Brand Personality

Calm, precise, and operator-friendly. The product should feel like a clean control desk for a sensitive autonomous system: quiet, trustworthy, and fast to scan.

## Anti-references

Avoid raw JSON walls as the primary interface, dark glass dashboards, gradient text, decorative observability visuals, English-heavy labels, marketing hero layouts, and component styles that make debugging feel theatrical.

## Design Principles

- Translate logs into decisions: show summaries, evidence, source, and impact before machine detail.
- Preserve operator confidence: every status should show whether it is live, dry-run, blocked, silent, or deliverable.
- Keep profile work reversible: config edit, validation, backup, and rollback should stay visible and understandable.
- Make the chain readable: World, State, Intent, Decision, Render, and Delivery should be visible as a single cause-and-effect path.
- Use Chinese first: labels, empty states, and action copy should match the operator's working language.

## Accessibility & Inclusion

Target practical WCAG AA contrast, keyboard reachable controls, visible focus states, reduced reliance on color alone, and motion limited to short state feedback.
